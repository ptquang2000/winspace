# 02.03 — Monitor model: `MonitorsChanged` fold + lookup + tests

**Labels:** `ready-for-agent`
**Depends on:** 02.01

## What to build

The pure-core monitor model the layout resolves `rcWork` against.

- `reduce(MonitorsChanged)`: **replace** `State.monitors` wholesale from the snapshot's
  `MonitorInfo` list (no incremental merge). Emits no Effect on its own.
- The layout `MonitorId → rcWork` lookup used by 02.02: a window whose `MonitorId` is absent
  from the map produces no fill (defensive — a window on an unknown monitor is left alone
  until the next topology snapshot).

## Acceptance criteria

- [ ] `MonitorsChanged` replaces the monitor topology wholesale (later snapshot wins entirely)
- [ ] After a snapshot, an `Appeared` resolves its `MonitorId` to that monitor's `rcWork` and
      fills there
- [ ] An `Appeared` whose `MonitorId` is not in the map emits no `PositionWindow`
- [ ] Tests drive `MonitorsChanged` + `Appeared` sequences and assert the resolved rect

## Out of scope

- Runtime `WM_DISPLAYCHANGE` reaction / re-tile (04)
- Per-Display fill order (04)
