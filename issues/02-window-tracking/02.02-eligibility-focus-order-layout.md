# 02.02 — Eligibility predicate + Focus order + layout + tests

**Labels:** `ready-for-agent`
**Depends on:** 02.01

## What to build

The behavioral heart of the slice, entirely in the pure Reducer.

- `constexpr bool isTileable(const WindowAttrs&)` — the Eligibility gate as the AND of the
  probed booleans: `topLevel && visible && thickFrame && caption && !toolWindow && !cloaked
  && !fullscreen`.
- `reduce(Appeared)`: classify. Ineligible ⇒ no Effect, no storage. Eligible ⇒ insert into
  Focus order at the front; **idempotent** on `WindowId` (already present ⇒ move-to-front,
  never duplicate). Then recompute layout.
- `reduce(Vanished)`: remove the `WindowId` from Focus order (no-op if absent). Then recompute
  layout — if the head changed, the survivor fills (reclaim).
- Layout function `layout(focusOrder, monitors) → effects`: if Focus order is non-empty, emit
  `PositionWindow` for `front()` filled to its monitor's `rcWork` (looked up by the window's
  `MonitorId`); empty ⇒ no Effect. This slice fills exactly the head; the function is the rail
  03 later swaps for BSP.

## Acceptance criteria

- [ ] `isTileable` returns the correct verdict; each condition individually flips it
- [ ] A single eligible `Appeared` emits exactly one `PositionWindow` to the correct `rcWork`
- [ ] A second eligible `Appeared` fills over the first (head = newest)
- [ ] `Vanished` of the head re-emits a fill for the survivor; the final `Vanished` emits none
- [ ] Two `Appeared` for the same `WindowId` ⇒ one Focus-order entry, one fill (idempotent)
- [ ] An ineligible `Appeared` emits **no** Effect and does not change Focus order
- [ ] Tests assert external behavior via emitted Effects (house style, `reducer_test.cpp`)

## Out of scope

- Focus reordering from `EVENT_SYSTEM_FOREGROUND` (03/05)
- Multi-window BSP layout (03)
