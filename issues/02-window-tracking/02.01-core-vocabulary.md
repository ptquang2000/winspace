# 02.01 — Core vocabulary: identities, Events, Effect, State

**Labels:** `ready-for-agent`
**Depends on:** 01, 11

## What to build

The pure-core plain-data vocabulary the rest of the slice consumes. No behavior yet — just
the types, in `winspace/reducer.cpp` (or a core header it includes), staying `windows.h`-free
and linker-provably pure (the test TU must still link no WM libraries).

- `enum class WindowId : uint64_t {}` and `enum class MonitorId : uint64_t {}` — strong,
  opaque identities. The core never does arithmetic on them.
- `struct Rect { int left, top, right, bottom; };` plain data.
- `struct WindowAttrs` — the Probe result: `id`, `monitor`, and the eligibility booleans
  (`topLevel, visible, thickFrame, caption, toolWindow, cloaked, fullscreen`).
- `struct MonitorInfo { MonitorId id; Rect rcWork; };`
- New `Event` alternatives: `Appeared{ WindowAttrs }`, `Vanished{ WindowId }`,
  `MonitorsChanged{ std::vector<MonitorInfo> }`.
- New `Effect` alternative: `PositionWindow{ WindowId id; Rect rect; }`.
- `State` additions: `monitors` (map `MonitorId → Rect`) and the Focus order (ordered
  container of `WindowId`). Keep pass-by-value/return-by-value.
- `operator==` for the new Events/Effects/`Rect` (the tests compare Effects by value, as the
  existing ones do).

## Acceptance criteria

- [ ] New identities, `Rect`, `WindowAttrs`, `MonitorInfo` defined in the pure core
- [ ] `Event` and `Effect` variants extended; the overloaded visitor in `reduce` remains
      exhaustive (a missing handler is a compile error)
- [ ] `State` carries the monitor map and Focus order; still trivially copyable/valued
- [ ] Value equality for the new plain-data types
- [ ] The test TU compiles and links with **no** WM import libraries (purity preserved)

## Out of scope

- Any reducer logic over the new types (02.02 / 02.03)
