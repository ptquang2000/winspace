# 02.04 — Hook adapter thread: probe, noise gate, transport, Adoption

**Labels:** `ready-for-agent`
**Depends on:** 02.02, 02.03

## What to build

A new I/O adapter (`io/window_hook.cpp`) that owns `<windows.h>` and turns the OS
accessibility stream into the pure-core Events from 02.01. Its own third thread, modeled on
the Hotkey thread (`hotkeyThreadMain` is the template); pulled into the app TU and spawned by
the spine (`io/app.cpp`) after the Worker HWND exists.

- One `SetWinEventHook` over `EVENT_OBJECT_CREATE … EVENT_OBJECT_UNCLOAKED`,
  `WINEVENT_OUTOFCONTEXT | WINEVENT_SKIPOWNPROCESS`. Callback is a free function reaching the
  worker HWND via `thread_local` (as the Events thread already does).
- First-pass noise gate: `idObject == OBJID_WINDOW && idChild == CHILDID_SELF && hwnd`.
- Edge mapping: `SHOW` / `UNCLOAKED` → **Probe** → post `Appeared`; `DESTROY` / `HIDE` /
  `CLOAKED` → post `Vanished`. Raw `CREATE` dropped.
- **Probe** = one synchronous read on the hook thread at event time, filling `WindowAttrs`:
  `GetWindowLong` styles/ex-styles, `DwmGetWindowAttribute(DWMWA_CLOAKED)`, top-level+unowned
  via `GetAncestor`/`GW_OWNER`, fullscreen via rect-vs-monitor, and `MonitorId` via
  `MonitorFromWindow`. All error handling uses the `io::Error` idiom from 11.
- Transport: heap `Event*` via the existing `postEvent(workerHwnd, …)` ownership contract.
- **Adoption:** at startup, one `EnumWindows` sweep posts a synthetic `Appeared` for each
  currently-shown top-level window (same path as live events) so winspace inherits the
  session. Also a startup `EnumDisplayMonitors` sweep posts one `MonitorsChanged`.
- Teardown: spine sends `PostThreadMessage(WM_QUIT)`; the thread `UnhookWinEvent`s and exits.

## Acceptance criteria

- [ ] Hook adapter runs on its own thread; spine spawns/joins it like the Hotkey thread
- [ ] Noise gate filters non-window accessibility events (carets, menus, scrollbars)
- [ ] `SHOW`/`UNCLOAKED`→`Appeared` (with a filled `WindowAttrs`), `DESTROY`/`HIDE`/`CLOAKED`
      →`Vanished`, `CREATE` dropped
- [ ] Startup Adoption sweep posts `Appeared` for existing windows; startup monitor sweep
      posts `MonitorsChanged`
- [ ] Clean teardown unhooks the win-event hook (no dangling system hook)
- [ ] Including `<windows.h>` here keeps the parser/Reducer linker-provably pure

## Out of scope

- Executing `PositionWindow` (02.05)
- `EVENT_SYSTEM_FOREGROUND` handling (03/05); `WM_DISPLAYCHANGE` handling (04)
