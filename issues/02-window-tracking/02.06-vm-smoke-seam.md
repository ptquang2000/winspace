# 02.06 — VM Smoke-seam steps for the live behaviors

**Labels:** `ready-for-agent`
**Depends on:** 02.05, 12

## What to build

Extend the VM seam-test harness (issue 12, ADR-0005) with the Smoke seams that only the
running binary can exhibit — the live behaviors the pure Reducer seam structurally cannot
reach. Each drives winspace on the guest desktop and asserts against an external Oracle (OS
window enumeration / geometry), not winspace's own log where an external observable exists.

Smoke seams to add:

- **Fill-one:** launch a normal app window; assert its rect equals the Display's `rcWork`
  after compensation (flush — extended-frame-bounds match the work area within tolerance).
- **Adoption:** with a window already open, start winspace; assert it fills that window.
- **Reclaim:** with two windows open, close the head; assert the survivor fills.
- **Ineligible left alone:** open a dialog / tool window; assert its rect is unchanged.
- **Cloaked UWP host excluded:** open a Store app; assert no phantom tile and the real window
  fills.
- **Clean unhook:** quit winspace; assert no dangling win-event hook / process.

## Acceptance criteria

- [ ] Each Smoke seam is expressed as a step in the harness's Manual smoke script format
- [ ] Assertions use OS state as the primary Oracle; stderr log only where no external
      observable exists
- [ ] Runs unattended on the existing VM snapshot via the Guest runner / Host orchestrator

## Out of scope

- Multi-window BSP parity (03's smoke steps)
- Multi-Display scenarios (04)
